The Virtual Machine

The Virtual Machine was created by me, after discussing the various Operating Systems and assessing my team's know-how on them I decided to go with Debian GNU/Linux.

Since the webapplication had to be hosted on a Virtual Machine we had to port forward certain services from the machine to it's host.
The Virtual Machine would not have a GUI so all development would take place on the host.

We could use VirtualBox's "Port Forwarding" option to port forward SSH, Apache and OpenLDAP for use on the host.
The most important service in terms of development was Samba, for this we used a Host-Only adapter, configured the interface on the guest and configured Samba's configuration to allow access to the webapplication's files for the developer user.

I distributed the Virtual Machine to my teammembers and got to work on the webapplication's framework.


The Framework

I decided to base our framework on MVC because of it's maintainability and ease of appending functionality.
After some research and following a few tutorials I got a decent framework set-up that would be the starting point of our project.

I created a Github repository, invited my team, we made some issues based on our already established requirements (mainly MoSCoW and Security Requirements) and distributed them.
